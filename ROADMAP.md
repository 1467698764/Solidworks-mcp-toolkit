# Roadmap

This roadmap focuses on practical SolidWorks + Codex workflows rather than raw API breadth. Items may change as real assemblies expose better priorities.

## Near term

- Capture a short live SolidWorks demo that shows inspect -> model-understand -> report-context -> worklog -> handoff-bundle -> audit.
- Add richer mate diagnostics from inspect reports and selection-report output.
- Add broader mechanical fixtures for locating features, fastening, sheet/plate interfaces, machined holes, clearances, and movable assemblies.
- Improve installer checks for Windows shells, Node, Python runtime selection, and MCP configuration copying.

## Future

- Optional C# bridge for SolidWorks API paths where COM automation from Python is fragile.
- More structured design-intent extraction for dimensions, configurations, and component patterns.
- Safer export presets for STEP/STL/PDF deliverables with reproducible manifests.
- Additional offline fixtures that represent messy real-world assembly naming and suppressed/hidden states.

## Non-goals

- No unattended bulk modification of user CAD files.
- No claim that automated checks replace engineering review, simulation, manufacturing review, or tolerance analysis.
- No public ranking claims; project positioning should stay evidence-based and practical.
